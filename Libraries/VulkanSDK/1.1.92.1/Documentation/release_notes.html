Vulkan SDK for Windows Release Notes

Vulkan SDK 1.1.92.1 Release Notes

Version 1.1.92.1 for Windows
November 27, 2018

This SDK supports Vulkan API revision 1.1.92. The prior SDK supported Vulkan API revision 1.1.85.

Much of the documentation for this SDK is contained in the Getting Started Guide, found in the Documentation directory in this SDK and on the LunarXchange website.

Drivers: LunarG recommends updating Vulkan-capable hardware drivers to the latest recommended version from the hardware vendor. A list of Known Issues appears at the end of this document with more information.

Overview of changes in SDK 1.1.92.1

This SDK fixes a loader bug when initializing the VK_EXT_debug_utils extension where the loader would pass in invalid instance handle to layers. This would cause a crash in layers that attempted to use the extension.

Overview of new features in SDK 1.1.92.0

  • vkconfig, the new Vulkan Configurator tool, is a graphical application that allows a user to specify which layers will be loaded by Vulkan applications at runtime--it provides an alternative to setting layers through environment variables or an application's layer selection. Additional information about Vulkan Configurator can be found here
  • VulkanSDK Ubuntu packages are no longer in beta. For instructions on how to access and install these packages, navigate to the SDK page on LunarXchange and select the "Ubuntu Packages" tab in the Linux section
  • Since the released of SDK 1.1.85.0, the following extensions have been added to the Vulkan spec and may be supported by a Vulkan device. To query extensions available to a given physical device, call vkEnumerateDeviceExtensionProperties() or contact the hardware vendor.
VK_EXT_calibrated_timestamps
VK_EXT_image_drm_format_modifier
VK_EXT_pci_bus_info
VK_EXT_transform_feedback
VK_GOOGLE_decorate_string
VK_AMD_memory_overallocation_behavior
VK_NV_ray_tracing
  • Bug fixes (see the SDK Components section below)
  • Improved validation coverage

Developer Notes

  • Executables cube and via are now named vkcube and vkvia respectively.
  • VUID validation coverage documentation is now included in the LunarG Vulkan SDK. This documentation (validation_error_database.csv and an HTML version) has been updated to represent the validation layer VUID coverage status as of this 1.1.92.0 SDK. validation_error_database.csv can be found locally in the SDK Documentation directory once the SDK is installed on the system. A spreadsheet editor can then be used to sort and view the data. Additionally, an HTML version of the validation error database can be found here.

Known Issues

  • The Vulkan Info application might not appear in the Start Menu if another version of the SDK is currently installed on the system. This can be corrected by uninstalling all other versions of the SDK.
  • Prior to the August 3, 2018 Steam update, applications such as Vulkan Cube could experience a crash in VkCreateSwapChainKHR() if the Steam Overlay Layer (VK_LAYER_VALVE_steam_overlay) was enabled. If this error occurs, first try updating the Steam client by clicking the "Steam" menu and selecting "Check for Steam Client Updates...". If the Steam client cannot be updated or the error persists, the behavior can be avoided by disabling the Steam Overlay Layer in one of the following ways:
    1. Set the DISABLE_VK_LAYER_VALVE_steam_overlay_1 environment variable equal to 1. This can be accomplished by editing the user environment variables under the Advanced System Properties menu or if a command prompt is utilized to execute the application, run the following command from the command prompt (Note: This change will only persist for a single command prompt session):
    set DISABLE_VK_LAYER_VALVE_steam_overlay_1=1
    1. Edit the Windows registry. To open the Windows registry, navigate to the Start Menu and either in the Run box or the Search box type regedit and press Enter. Navigate to Computer -> HKEY_LOCAL_MACHINE_SOFTWARE -> SOFTWARE -> Khronos -> Vulkan -> ImplicitLayers. The SteamOverlayVulkanLayer64.json entry should be found here. Double click the file name and change the value in the "Value Data" box to 1. For 64-bit versions of Windows, an additional step must be performed by making the same change in the WOW6432Node directory. The registry entries found in the WOW6432Node directory are referenced when 32-bit programs are run on a 64-bit system. To make this change, navigate to Computer -> HKEY_LOCAL_MACHINE_SOFTWARE -> SOFTWARE -> WOW6432Node -> Khronos -> Vulkan -> ImplicitLayers. For the SteamOverlayVulkanLayer.json entry, double-click the file name and change the value in the "Value Data" box to 1.
